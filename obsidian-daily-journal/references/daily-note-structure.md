---
title: Daily Note Structure
---

The daily note is used to capture different types of information throughout the day. When processing random thoughts, categorize them into one of the following sections:

## ## Technical Notes & Snippets
Use this section for:
- Code snippets or technical insights.
- Brainstorming software architecture.
- Random technical discoveries.
- Troubleshooting steps for a specific problem.

## ## Reflections
Use this section for:
- Personal thoughts and "fleeting thoughts" about the day.
- General musings that aren't strictly technical.
- Career or personal growth observations.

## ## Daily Focus
Use this section for:
- Important tasks or goals for the day.
- Brief updates on progress.

## ## Meetings
Use this section for:
- Summaries or points from meetings.

## ## Ramblings
Use this section for:
- Stream-of-consciousness thoughts, half-baked ideas, and unstructured brain dumps.
- Anything that does not yet fit a category. No formatting required.
- The `idea-extractor`, `proposal-drafter`, and `task-scaffolder` skills can later process these into structured Idea, Proposal, Task, or Feature notes.

## Formatting Guidelines
- Each entry should start with a bullet point `- `.
- If a thought is a continuation or a sub-point, indent it accordingly.
- Use Markdown formatting (bold, italics, code blocks) as appropriate.
